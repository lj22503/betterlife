# 体验记忆追踪器 📸

**基于《最优解人生》（Die With Zero）核心理念**

---

## 📋 功能描述

记录用户的人生体验，追踪回忆红利，生成体验报告，帮助用户最大化体验价值。

**适用场景：**
- 记录已完成体验
- 追踪回忆红利积累
- 生成年度体验报告
- 识别体验模式和问题

**边界条件：**
- 内部技能，非对外服务
- 聚焦记录和追踪，非规划
- 数据存储在本地

---

## 🎯 核心功能

### 1. 体验记录

记录已完成的体验，包含详细信息。

**记录字段：**
- 体验名称
- 日期/时长
- 地点
- 同伴
- 花费
- 愉悦度评分（1-10）
- 照片/视频链接
- 文字记录

**输出：**
```json
{
  "experience_id": "exp_20260318_001",
  "name": "西藏徒步",
  "date": "2026-03-15",
  "duration_days": 7,
  "location": "西藏林芝",
  "companions": ["配偶", "好友 A"],
  "cost": 25000,
  "joy_score": 9,
  "media": ["photo_001.jpg", "video_001.mp4"],
  "notes": "最美的桃花季，体力挑战大但值得"
}
```

### 2. 回忆红利追踪

计算体验产生的回忆红利。

**公式：**
```
回忆红利 = 愉悦度 × 已过年数 × 分享次数
```

**输出：**
```json
{
  "experience": "西藏徒步",
  "joy_score": 9,
  "years_passed": 0,
  "shares": 50,
  "current_dividend": 0,
  "projected_dividend_10y": 9 × 10 × 200 = 18000,
  "status": "红利积累中"
}
```

### 3. 年度体验报告

生成年度体验总结报告。

**报告内容：**
- 本年度体验数量
- 总花费
- 平均愉悦度
- 最佳体验 TOP3
- 体验类别分布
- 与往年对比

**输出：**
```markdown
# 2026 年度体验报告

## 概览
- 体验数量：12 次
- 总花费：15 万元
- 平均愉悦度：8.2/10

## 最佳体验 TOP3
1. 西藏徒步（9/10）- 2.5 万
2. 潜水考证（9/10）- 1.8 万
3. 家庭旅行（8.5/10）- 3 万

## 类别分布
- 旅行：5 次（42%）
- 学习：3 次（25%）
- 运动：2 次（17%）
- 社交：2 次（17%）

## 对比 2025
- 体验数量：+2 次
- 总花费：+3 万
- 平均愉悦度：+0.5
```

### 4. 体验模式分析

分析用户的体验模式，识别问题。

**分析维度：**
- 体验频率（是否过少/过多）
- 体验多样性（是否过于单一）
- 支出合理性（是否过低/过高）
- 愉悦度趋势（是否下降）

**输出：**
```json
{
  "patterns": {
    "frequency": "低（年均 5 次，建议 12 次）",
    "diversity": "中（旅行占比 80%）",
    "spending": "低（年收入 5%，建议 10-20%）",
    "joy_trend": "稳定（8.0-8.5）"
  },
  "issues": [
    "体验频率过低，建议增加周末短途",
    "体验类型单一，建议增加学习/社交类"
  ],
  "suggestions": [
    "每月至少 1 次新体验",
    "尝试从未做过的活动类型"
  ]
}
```

- 完整的年度体验报告（Markdown 格式）
- 可分享到社交媒体

## ⚠️ 错误处理

| 错误码 | 含义 | 处理建议 |
|--------|------|----------|
| 400 | 参数缺失 | 检查必填字段（action、user_id） |
| 400 | 体验数据不完整 | 记录体验时需要完整字段 |
| 404 | 体验不存在 | 检查体验 ID 是否正确 |
| 500 | 内部错误 | 重试或联系开发者 |
