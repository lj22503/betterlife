# 习惯追踪 📊

**基于《原子习惯》**

---

## 📋 功能描述

指导用户通过习惯追踪可视化进步，保持动力，实践"绝不错过两次"原则。

**适用场景：**
- 习惯坚持困难
- 缺乏反馈
- 动力不足
- 进步不明显

**边界条件：**
- 不替代内在动机
- 避免过度执着
- 个体差异需调整

---

## 🎯 核心功能

### 1. 习惯追踪原理

**为什么追踪有效：**
```
心理机制：
✓ 可视化进步
✓ 即时满足感
✓ 建立成就感
✓ 强化身份认同

行为科学：
✓ 第四定律（令人愉悦）
✓ 即时奖励
✓ 反馈循环
✓ 承诺一致性
```

**追踪的好处：**
```
• 让进步可见
• 提供即时奖励
• 建立连续记录
• 强化身份证据
• 发现模式问题
• 保持动力
```

### 2. 习惯追踪方法

**方法 1：日历打勾法**
```
工具：
• 挂历/台历
• 红色马克笔
• 每天完成后打勾✓

优势：
✓ 简单直观
✓ 视觉冲击强
✓ 连续记录明显
✓ 不想打破链条

实践：
• 放在显眼位置
• 每天固定时间打勾
• 享受链条延长
```

**方法 2：习惯追踪 APP**
```
推荐功能：
✓ 习惯列表
✓ 每日打卡
✓ 连续天数统计
✓ 进度图表
✓ 提醒功能

优势：
✓ 方便携带
✓ 自动统计
✓ 图表可视化
✓ 社交分享

注意：
× 不要过度依赖
× 保持简单
× 重点在行动
```

**方法 3：纸质追踪表**
```
模板：
• 30 天表格
• 习惯列表
• 每日打勾
• 月度总结

优势：
✓ 可定制
✓ 无电子干扰
✓ 仪式感强
✓ 可回顾

实践：
• 打印或手绘
• 放在显眼处
• 每天填写
```

**方法 4：习惯叠加 + 追踪**
```
公式：
"在［当前习惯］之后，我将［追踪我的习惯］"

例子：
• 喝完咖啡后，我将在日历上打勾
• 刷牙后，我将在 APP 上打卡
• 睡觉前，我将在表格上记录

优势：
✓ 自动化
✓ 与现有习惯绑定
✓ 减少遗忘
```

### 3. "绝不错过两次"原则

**原则核心：**
```
核心理念：
• 错过一天没关系
• 但绝不错过连续两天
• 快速恢复是关键

为什么重要：
✓ 一次错过是意外
✓ 两次错过是新习惯开始
✓ 快速恢复防止滑坡
✓ 保持身份认同
```

**实践策略：**
```
预防错过：
✓ 设置提醒
✓ 降低标准（两分钟版本）
✓ 准备备用方案
✓ 环境提示

错过后的恢复：
✓ 不自责
✓ 立即恢复
✓ 降低难度
✓ 记住身份
```

**错过处理流程：**
```
步骤 1：接受错过
• 不批判
• 不自责
• 承认是人之常情

步骤 2：分析原因
• 什么导致错过？
• 提示不够明显？
• 阻力太大？
• 奖励不足？

步骤 3：立即恢复
• 今天就做
• 两分钟版本也可以
• 重新打勾
• 继续链条
```

```
进度条型：

计数型：
• 连续天数
• 总次数
• 累计时间

视觉型：
• 日历链条
• 填充图表
• 颜色变化
```

**设计原则：**
```
✓ 简单明了
✓ 即时更新
✓ 视觉冲击
✓ 易于维护
✓ 与身份连接

避免：
× 过于复杂
× 难以维护
× 压力过大
× 完美主义
```

### 5. 习惯追踪实践

**每日追踪例程：**
```
早晨：
✓ 查看今日习惯
✓ 设置提醒
✓ 准备环境

执行时：
✓ 专注行动
✓ 享受过程
✓ 不执着结果

完成后：
✓ 立即追踪
✓ 享受打勾
✓ 庆祝小赢

睡前：
✓ 检查完成情况
✓ 准备明天
✓ 感恩进步
```

**每周回顾：**
```
问题：
1. 本周完成了多少？
2. 什么帮助了我？
3. 什么阻碍了我？
4. 下周如何改进？

行动：
✓ 庆祝成功
✓ 调整策略
✓ 设定下周目标
```

**每月总结：**
```
回顾：
• 月度完成率
• 连续天数记录
• 最大挑战
• 最大收获

反思：
• 什么有效？
• 什么无效？
• 身份有何变化？
• 下月重点？
```

### 6. 常见问题解答

**问题 1：追踪太麻烦怎么办？**
```
解决：
✓ 简化追踪方式
✓ 习惯叠加追踪
✓ 使用 APP 自动化
✓ 两分钟追踪版本

关键：
• 追踪本身也是习惯
• 从简单开始
• 找到适合的方式
```

**问题 2：错过几天后想放弃怎么办？**
```
解决：
✓ 记住"绝不错过两次"
✓ 降低标准（两分钟）
✓ 回顾身份认同
✓ 从小赢重新开始

关键：
• 完美是进步的敌人
• 恢复比完美重要
• 每次行动都是投票
```

**问题 3：追踪变成压力怎么办？**
```
解决：
✓ 降低期望
✓ 关注过程而非结果
✓ 允许弹性
✓ 记住为什么开始

关键：
• 追踪是工具不是目的
• 保持灵活性
• 享受过程
```
